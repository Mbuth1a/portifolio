## Deployment Instructions

### Option 1: GitHub Pages (Free)

1. **Create a new repository:**
   - Go to GitHub.com and sign in
   - Click the "+" icon → "New repository"
   - Repository name: `portifolio` (or your preferred name)
   - Select "Public"
   - Click "Create repository"

2. **Upload files:**
   - On your repository page, click "uploading an existing file"
   - Drag and drop all your files (index.html, css/, js/, images/)
   - Click "Commit changes"

3. **Enable GitHub Pages:**
   - Go to repository "Settings"
   - Click "Pages" in the left sidebar
   - Under "Build and deployment" → "Source", select "Deploy from a branch"
   - Under "Branch", select "main" (or "master") and "/ (root)"
   - Click "Save"
   - Wait 1-2 minutes for deployment

4. **Your site will be live at:**
   
```
   https://mbuth1a.github.io/portifolio/
   
```

### Option 2: Netlify (Free Alternative)

1. Go to Netlify.com and sign up with GitHub
2. Click "Add new site" → "Import an existing project"
3. Select your GitHub repository
4. Click "Deploy site"

Your site will be live at a random Netlify subdomain (you can customize it).

### Option 3: Vercel (Free Alternative)

1. Go to Vercel.com and sign up with GitHub
2. Click "Add New Project" → "Import Project"
3. Select your GitHub repository
4. Click "Deploy"

## Custom Domain (Optional)

If you have a custom domain:
- For GitHub Pages: Go to Settings → Pages → Custom domain
- For Netlify/Vercel: Go to Domain Settings in your dashboard
