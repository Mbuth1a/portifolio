Lab Challenges

CTF challenges and security labs I've completed

Configure and Verify a Site-to-Site IPsec VPN using CLI

Easy

Problem Statement

Configure R1 and R3 routers to establish a site-to-site IPsec VPN tunnel for secure communication between their LANs through R2.

Approach

I started by analyzing the network topology and identifying the routers involved (R1, R2, R3). Then I configured ISAKMP (IKE) policies on both R1 and R3 to establish the VPN tunnel. Next, I created transform sets to define the encryption and hash algorithms. I configured crypto maps to bind the ISAKMP policies with the tunnel endpoints. Finally, I applied the crypto maps to the router interfaces and verified the VPN tunnel status.

Tools Used

Cisco IOS CLI Wireshark Ping Show Commands

Screenshots

Key Lessons Learned

  • Understanding the difference between IKEv1 and IKEv2 protocols for VPN establishment
  • Learning how to configure transform sets for encryption (ESP) and authentication (SHA/HMAC)
  • Troubleshooting VPN tunnels using show commands and debug outputs
  • Understanding the role of NAT traversal in IPsec VPN configuration

Switch Security Configuration

Medium

Problem Statement

Configure VLANs on switches and implement switch security features including port security, DHCP snooping, and ARP inspection.

Approach

I began by creating and configuring VLANs on the switches to segment network traffic. Then I configured VLAN trunking protocol (VTP) for VLAN information exchange between switches. Next, I implemented port security features including MAC address limiting and violation actions. I configured DHCP snooping to prevent rogue DHCP servers and enabled Dynamic ARP Inspection (DAI). Finally, I tested the security configurations by attempting various attacks and verifying the defenses.

Tools Used

Cisco Packet Tracer Ping Show Commands CLI

Screenshots

Key Lessons Learned

  • Learning how to properly configure VLANs and trunk ports for network segmentation
  • Implementing port security to prevent unauthorized access and MAC address spoofing
  • Understanding DHCP snooping to protect against DHCP starvation attacks
  • Configuring Dynamic ARP Inspection to prevent ARP spoofing and man-in-the-middle attacks

Configure ASA Basic Settings and Firewall Using the CLI

Hard

Problem Statement

Configure Cisco ASA with basic settings, NAT/PAT, access lists, and DHCP to secure the corporate network and DMZ from the ISP.

Approach

I started by configuring basic ASA settings including hostname, domain name, and clock settings. Then I configured interfaces with appropriate security levels and IP addresses. Next, I set up NAT/PAT to translate internal private addresses to public IPs. I configured access lists to allow specific traffic flows between zones (inside, DMZ, outside). I implemented DHCP services for internal hosts and set up remote management access. Finally, I tested all connectivity and security policies to ensure proper operation.

Tools Used

Cisco ASA CLI Packet Tracer Ping Show Commands Telnet/SSH

Screenshots

Key Lessons Learned

  • Understanding ASA security zones and interface security levels
  • Learning NAT/PAT configuration for address translation
  • Implementing access control lists (ACLs) for firewall policies
  • Configuring DHCP services and remote management access on ASA
  • Troubleshooting connectivity issues through ASA
